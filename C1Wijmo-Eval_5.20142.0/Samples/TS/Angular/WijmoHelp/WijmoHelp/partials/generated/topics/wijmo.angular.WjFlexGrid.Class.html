AngularJS directive for the FlexGrid control.

Use the WjFlexGrid directive to add grids to your AngularJS applications. Note that directive and parameter names must be formatted as lower-case with dashes instead of camel-case. For example:

<p>Here is a FlexGrid control:</p>
<wj-flex-grid items-source="data">
  <wj-flex-grid-column header="Country" binding="country"></wj-flex-grid-column>
  <wj-flex-grid-column header="Sales" binding="sales"></wj-flex-grid-column>
  <wj-flex-grid-column header="Expenses" binding="expenses"></wj-flex-grid-column>
  <wj-flex-grid-column header="Downloads" binding="downloads"></wj-flex-grid-column>
</wj-flex-grid>

The example below creates a FlexGrid control and binds it to a 'data' array exposed by the controller. The grid has three columns, each corresponding to a property of the objects contained in the source array.

The WjFlexGrid directive supports the following attributes:

allowAddNew
@ Whether to show a new row template so users can add items to the source collection.
allowDelete
@ Whether the grid should delete selected rows when the user presses the Delete key.
allowDragging
@ Whether rows and/or columns can be dragged with the mouse.
allowMerging
@ Determines which parts of the grid provide cell merging.
allowResizing
@ Whether users are allowed to resize rows and/or columns with the mouse.
allowSorting
@ Whether users are allowed to sort columns by clicking the column headers.
autoGenerateColumns
@ Whether the grid should generate columns automatically based on the itemsSource.
childItemsPath
@ Name of the property used to generate child rows in hierarchical grids.
control
= Reference to the FlexGrid control created by this directive.
groupHeaderFormat
@ Format string used to create the group header content.
headersVisibility
= Whether the row and column headers are visible. (see HeadersVisibility).
itemFormatter
= Formatter function used to customize cells on this grid.
itemsSource
= Array or ICollectionView that contains items shown on the grid.
isReadOnly
@ Whether the user can edit the grid cells by typing into them.
mergeManager
= Merge Manager (see MergeManager).
scrollPosition
= Represents the value of the grid's scrollbars.
selection
= Current selection (see CellRange).
selectionMode
@ Selection mode (see SelectionMode).
showGroups
@ Whether to insert group rows to delimit data groups.
showSort
@ Whether to display sort indicators in the column headers.
treeIndent
@ Indentation used to offset row groups of different levels.
beginningEdit
& The beginningEdit event handler (see beginningEdit).
cellEditEnded
& The cellEditEnded event handler (see cellEditEnded).
cellEditEnding
& The cellEditEnding event handler (see cellEditEnding).
prepareCellForEdit
& The prepareCellForEdit event handler (see prepareCellForEdit).
resizingColumn
& The resizingColumn event handler (see resizingColumn).
resizedColumn
& The resizedColumn event handler (see resizedColumn).
draggedColumn
& The draggedColumn event handler (see draggedColumn).
draggingColumn
& The draggingColumn event handler (see draggingColumn).
sortedColumn
& The sortedColumn event handler (see sortedColumn).
sortingColumn
& The sortingColumn event handler (see sortingColumn).
deletingRow
& The deletingRow event handler (see deletingRow).
draggingRow
& The draggingRow event handler (see draggingRow).
draggedRow
& The draggedRow event handler (see draggedRow).
resizingRow
& The resizingRow event handler (see resizingRow).
resizedRow
& The resizedRow event handler (see resizedRow).
rowAdded
& The rowAdded event handler (see rowAdded).
rowEditEnded
& The rowEditEnded event handler (see rowEditEnded).
rowEditEnding
& The rowEditEnding event handler (see rowEditEnding).
loadedRows
& The loadedRows event handler (see loadedRows).
loadingRows
